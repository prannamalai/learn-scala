Call by Value / Name

Evaluation Strategies

    Name is evaluated by replacing the right hand side of the definition

    .... till it evaluates to a value

    Let's look at substituition model

				  
def sumOfSquares(x:Int, y:Int)= square(x) + square (y)
					  
					  
def test(x:Int, y:Int) = x * x

Call by Value / Name

    Call by Value has the advantage of evaluating function argument only once

    Call by Name has the advantage of not evaluating function argument if not used

def vs val

    def evaluates on every performance call

    def creates a new function on every call

    def allows type parametrization

    val evaluates when defined

val vs. var

  • Everything in Scala is an expression (Almost)
  • You can assign the result of an expression to a name
  • You have two options
    1. Use val - will not change the binding
    2. Use var - can change the binding
  • Let's experiment in REPL

Functions

  • A Function is a mapping that takes one or more arguments and produces a single result.
  • Create functions with the def keyword
  • You need type signature for paramaters
  • You don't need paranthesis for functions without arguments
  • Interepreter will happily return the signature type ... mostly
  • Let's experiment in REPL
